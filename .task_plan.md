Qwen 27B/35B coders all offload and run at ~2.3-5 tok/s), stage it, and
benchmark it on-box.

## Findings
- Qwen3.5-9B-MTP: ~1.65x decode speedup over non-MTP base
  (~98-104 vs ~58-63 tok/s, graphs-off).  In Unsloth's claimed 1.5-2x
  range.  Default coding driver should be qwen3.5-9b-mtp-coding.
- MTP + CUDA graphs = hard crash (illegal memory access).  Same family
  of issue as Qwen3.6-27B-MTP in session_summary_runtime_ab.md.
  Production already runs graphs-off via systemd, so unaffected.
- Qwopus shows no throughput edge over base 9B (~60 vs ~60 tok/s) -
  expected for a finetune.  Its coding-quality claim still untested
  (throughput run doesn't address it).

## Open questions
- Does Qwopus3.5-9B-Coder actually beat the base Qwen3.5-9B on coding?
  Edge is self-reported; needs a coding-quality rubric, not throughput.
