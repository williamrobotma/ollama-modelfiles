# Task Plan: Migrate off Ollama-managed GGUFs -> local HF-cache files

Branch: `migrate/hf-local-ggufs` (off `refactor/benchmark-common-harness`).
Predecessor plan (9B-coders benchmark) completed; see git history and
session_summary_qwen3.5_9b_mtp_bench.md.

## Objective
Every Modelfile references GGUFs provisioned by `hf download` (HF cache,
llama.cpp-convention files) instead of `hf.co/...` OCI pulls or
registry.ollama.ai tags. Rationale: the same files then serve both Ollama
(`FROM /abs/path.gguf`) and llama.cpp directly (`--model/--model-draft`),
the OCI config-blob hang class of failure disappears, and Gemma MTP
(target + separate drafter; llama.cpp-only on CUDA) becomes runnable
from the same cache.
